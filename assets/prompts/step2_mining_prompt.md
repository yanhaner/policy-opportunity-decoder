# 步骤2：需求挖掘提示词

> 用于指导AI完成政策文件分析的第二步——需求挖掘与痛点诊断

---

## 任务目标

穿透政策表面，洞察政策执行者（政府、平台公司等）的显性与隐性需求，形成《目标客户痛点分析与需求转化表》。

---

## 分析框架

### 1. 需求冰山模型

```
        /\
       /  \\      【显性需求】
      /____\\       政策明确要求完成的任务
     /      \\
    /        \\    【隐性需求】
   /__________\\    执行中的真实痛点
  /            \\
 /              \\ 【深层需求】
/________________\\ 政治/经济/社会综合诉求
```

### 2. 客户画像维度

| 维度 | 分析内容 |
|-----|---------|
| **客户类型** | 政府部门/平台公司/其他 |
| **决策层级** | 决策层/管理层/执行层 |
| **核心关切** | 政治账/经济账/民生账 |
| **考核指标** | KPI/年度任务/任期目标 |
| **痛点问题** | 当前面临的主要困难 |
| **预算能力** | 资金充裕度/来源渠道 |
| **决策流程** | 决策链条/关键节点 |
| **信息渠道** | 获取信息的途径和偏好 |

### 3. 痛点分类

#### 按性质分类
| 类型 | 描述 | 示例 |
|-----|------|------|
| **资源型痛点** | 缺乏必要的资源 | 资金不足、人才短缺、土地紧张 |
| **能力型痛点** | 缺乏必要的能力 | 技术能力不足、管理经验缺乏 |
| **信息型痛点** | 信息不对称 | 政策不了解、市场不清楚 |
| **协调型痛点** | 协调困难 | 部门协调难、居民协调难 |
| **风险型痛点** | 担心风险 | 怕担责、怕出事、怕投诉 |

#### 按层级分类
| 层级 | 描述 | 关注点 |
|-----|------|-------|
| **决策层痛点** | 战略层面的担忧 | 政绩、风险、形象 |
| **管理层痛点** | 执行层面的困难 | 进度、质量、成本 |
| **执行层痛点** | 操作层面的问题 | 方法、标准、配合 |

## 分析要点

### DO（应该做）

- [ ] 站在客户角度思考问题
- [ ] 区分不同层级客户的需求
- [ ] 挖掘水面下的隐性痛点
- [ ] 理解客户的决策逻辑
- [ ] 将痛点转化为价值点
- [ ] 验证痛点的真实性

### DON'T（不应该做）

- [ ] 不要只关注显性需求
- [ ] 不要忽视客户的深层诉求
- [ ] 不要假设客户知道解决方案
- [ ] 不要低估客户的预算能力
- [ ] 不要忽视竞争对手的影响

## 示例

### 背景

政策任务：建设高标准农田100万亩
目标客户：某县农业农村局

### 输出

**客户画像**：

| 维度 | 描述 |
|-----|------|
| **客户类型** | 县级农业农村部门 |
| **决策层级** | 管理层（局长、分管副局长） |
| **核心关切** | 任务完成、资金安全、农民满意 |
| **考核指标** | 建设面积、工程质量、验收合格率 |
| **痛点问题** | 资金缺口大、技术标准杂、协调难度大 |
| **预算能力** | 中等，依赖中央和省级资金 |

**需求冰山分析**：

- **显性需求**：完成100万亩高标准农田建设任务
- **隐性需求**：
  - 资金缺口如何弥补
  - 技术标准如何选择
  - 乡镇和农民如何协调
  - 工程质量如何保证
- **深层需求**：
  - 打造农业工作亮点
  - 获得上级表彰
  - 避免审计问题
  - 确保农民满意不出事

**痛点分析与需求转化表**：

| 客户任务 | 显性痛点 | 隐性痛点 | 解决方案价值点 |
|---------|---------|---------|---------------|
| 筹措建设资金 | 财政资金不足 | 担心增加县级债务 | 设计多元化融资方案，整合各级资金 |
| 选择技术标准 | 标准多、更新快 | 担心选择不当被问责 | 提供标准解读和技术指导服务 |
| 协调乡镇农民 | 征地协调难 | 担心引发群体性事件 | 提供全程协调服务，建立利益共享机制 |
| 保证工程质量 | 施工队伍素质参差不齐 | 担心验收不通过 | 提供全过程工程咨询服务 |

---

## 执行指令

请基于步骤1识别的政策机会点，对目标客户进行深度需求挖掘，输出：

1. 目标客户画像（至少2类客户）
2. 需求冰山分析（显性/隐性/深层需求）
3. 目标客户痛点分析与需求转化表（至少20个任务）

分析过程中，如有需要澄清的信息，请及时提出。
