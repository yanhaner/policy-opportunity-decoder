# 步骤1：政策解码提示词

> 用于指导AI完成政策文件的第一步分析——政策解码与机会扫描

---

## 任务目标

像"手术刀"一样拆解政策，提取关键信号和项目线索，形成《政策核心要素与机会点清单》。**必须逐章遍历全文，严禁遗漏任何包含实质性业务机会的章节（如产业、文化、民生、生态、基建等）！**

---

## 分析框架

### 1. 层级定位

分析政策文件的层级和性质：

| 分析项 | 内容 |
|-------|------|
| **文件层级** | 国家级/省级/市级/区县级 |
| **文件性质** | 规划纲要/实施意见/工作方案/技术规范 |
| **法律效力** | 强制性/指导性/参考性 |
| **目标区域** | 政策适用的地理范围 |

### 2. 关键词提取

从以下维度提取关键词：

#### 发展方向关键词
- 识别文件中关于"发展方向"的核心词汇
- 记录出现频次和重要程度

#### 重点任务关键词
- 识别"重点任务""重点工作"等章节内容
- 提取量化指标和时间节点

#### 支持领域关键词
- 识别"支持""鼓励""引导"等表述
- 明确支持的具体领域和方式

#### 资金投向关键词
- 识别资金来源和投向
- 记录资金规模和申请方式

#### 约束条件关键词
- 识别"禁止""限制""必须"等强制性表述
- 明确准入条件和负面清单

### 3. 机会映射

使用"政策-资金-项目转化漏斗"思维：

```
政策条款（宏观导向）
    ↓
资金支持（中观渠道）
    ↓
项目机会（微观落地）
```

提炼出具体的“机会方向”。将条款映射到研究类、规划类、咨询类、设计类、数字化、投融资、运营类等项目线索。

## 分析要点

### DO（应该做）

- [ ] **逐章、逐板块完整遍历政策全文，确保全景式覆盖**
- [ ] 重点核查是否提取了产业体系、文化建设、文旅融合、社会民生、生态治理、基础设施等极易被跳过的重要章节机会
- [ ] 区分"方向性表述"和"具体任务"
- [ ] 识别量化指标和时间节点
- [ ] 关注资金支持和申报方式
- [ ] 结合目标区域实际情况分析
- [ ] 使用表格结构化呈现信息

### DON'T（不应该做）

- [ ] **绝对不要只挑选前几章或少数几章进行解读而遗漏全文的其他重要部分！**
- [ ] 不要过度解读方向性表述
- [ ] 不要忽视政策的约束条件
- [ ] 不要脱离目标区域实际
- [ ] 不要遗漏资金支持信息
- [ ] 不要混淆不同层级的政策

---

## 机会等级判定标准

| 等级 | 判定标准 | 后续行动 |
|-----|---------|---------|
| **高** | 政策明确+资金保障+客户清晰+业务匹配 | 重点跟进 |
| **中** | 政策较明确+资金可能+客户可找+业务相关 | 持续跟踪 |
| **低** | 政策模糊+资金不明+客户不清+业务较远 | 关注即可 |

## 执行指令

请根据上述框架，对提供的政策文件进行系统分析，输出：

1. 政策基本信息
2. 政策层级定位
3. 核心内容梳理（发展目标、重点任务、支持政策）
4. **政策核心要素与机会点清单（【核心约束】必须覆盖政策全文的每一个主要章节或板块，严禁自行删减跳过！确保产业、基建、文旅、生态、民生等板块机会一览无余）**

分析过程中，如有需要澄清的信息，请及时提出。
